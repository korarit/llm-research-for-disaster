## 2. การวิเคราะห์ผลการทำ Ensemble Voting (ดีขึ้นหรือไม่?)

### 1. 🎯 ด้านการกรองความเกี่ยวข้อง (Informativeness Classification)
* **ปรับปรุงดีขึ้นเล็กน้อยในโมเดลความสามารถใกล้เคียงกัน (Exp 3E)**:
  * ในกลุ่ม **Exp 3E** โมเดลเดี่ยวทั้งสามตัวมีประสิทธิภาพเกาะกลุ่มใกล้เคียงกันมาก (`0.81 - 0.84`) การนำมาทำ Ensemble Voting ช่วยให้เกิด Synergy Effect ดึงคะแนน F1 สูงกว่าโมเดลเดี่ยวได้ในบางจุด เช่น ที่ Temp 0.1 ได้ F1 = **0.8428** (ชนะ Deepseek ที่ทำได้สูงสุดเดี่ยวที่ 0.8405) และ Temp 0.2 ได้ F1 = **0.8380** (ชนะ Deepseek เดี่ยวที่ 0.8347)
* **โดนโมเดลที่ประสิทธิภาพต่ำดึงลง (Exp 3 และ 3F)**:
  * ใน **Exp 3** และ **Exp 3F** เนื่องจากมีโมเดลใดโมเดลหนึ่งตกลงมามากผิดปกติ (เช่น Typhoon ใน Exp 3 และ Exp 3F ที่ได้ F1 ต่ำกว่าเพื่อนราว -0.08) ส่งผลให้เสียงส่วนใหญ่โหวตพลาดตามไปด้วย ทำให้คะแนน Ensemble ร่วงต่ำกว่าโมเดลเดี่ยวที่ดีที่สุดเฉลี่ยประมาณ **-0.01 ถึง -0.03**

### 2. 🗂️ ด้านการระบุหมวดหมู่ภัยพิบัติ (Category Classification)
* **การชนะโมเดลเดี่ยวใน Exp 3E**:
  * เป็นที่น่าสนใจอย่างยิ่งว่า ในกลุ่ม **Exp 3E** คะแนน F1 Category ของ Ensemble Voting สูงกว่าโมเดลเดี่ยวที่ดีที่สุดในหลายอุณหภูมิ เช่น ที่ Temp 0.1 (F1 = **0.5921** เทียบกับ Gemma เดี่ยวสูงสุดที่ 0.5783 หรือ **+0.0137**) และที่ Temp 0.3 (F1 = **0.5930** เทียบกับ Gemma ที่ 0.5870 หรือ **+0.0060**)
  * นี่เกิดจากระดับความสามารถของทั้งสามโมเดลบนสถาปัตยกรรมแบบ Sequential นี้มีความใกล้เคียงกันมาก การทำ Ensemble จึงช่วยกรองความคลาดเคลื่อนแบบสุ่มของแต่ละโมเดลออกไปได้อย่างมีนัยสำคัญ

## 3. ข้อจำกัดและข้อควรพิจารณาในการใช้งานจริง (Trade-offs)

1. **ปริมาณโทเค็นที่ใช้สูงมากเป็นประวัติการณ์ (Extremely Severe Token Overheads)**:
   * เนื่องจากระบบ Sequential 2-Agent มีการแบ่ง API Request ออกเป็นสองขั้นตอน ซึ่งทำให้เสีย Token มากกว่าระบบ Flat หรือ Joint อยู่แล้ว
   * เมื่อนำทั้ง 3 โมเดลมารันเพื่อทำ Ensemble ปริมาณการใช้งานโทเค็นจะคูณ 3 เข้าไปอีก 
   * ตัวอย่างเช่น ใน Exp 3E หากรัน Ensemble Voting จะใช้โทเค็นรวมทะลุถึง **~1,692,000 โทเค็น** ต่อรอบ ซึ่งสูงมากเมื่อเทียบกับประสิทธิภาพ F1 Category ที่ระดับ `0.59`

2. **คำแนะนำเชิงกลยุทธ์**:
   * แม้การทำ Ensemble ใน Exp 3E จะให้ผลลัพธ์ที่ดีขึ้นกว่าโมเดลเดี่ยวของตัวมันเอง แต่คะแนน F1 Category สูงสุดที่ทำได้ (**0.5930**) ก็ยัง **ต่ำกว่าการใช้โมเดลเดี่ยว** บนสถาปัตยกรรมอื่นอย่างชัดเจน (เช่น `typhoon-v2.5` เดี่ยวใน Exp 2E ที่ทำคะแนนได้สูงถึง **0.6493** หรือ `typhoon-v2.5` เดี่ยวใน Exp 1E ที่ทำได้ **0.6197**)
   * ดังนั้น ในสถาปัตยกรรม Sequential **ไม่แนะนำให้ทำ Ensemble Voting ในทางปฏิบัติ** เนื่องจากสิ้นเปลืองงบประมาณมากเกินความจำเป็น และไม่สามารถให้ผลสัมฤทธิ์ปลายทางที่สูงสู้โมเดลเดี่ยวบนสถาปัตยกรรม Flat/Joint ได้
