# รายงานเปรียบเทียบผลลัพธ์การคัดแยกข้อความภัยพิบัติทุกเวอร์ชัน (V1 vs. V2 vs. V2-1)

รายงานฉบับนี้วิเคราะห์เปรียบเทียบความแตกต่างด้านประสิทธิภาพการคัดแยกประเภทความช่วยเหลือ (Category F1-Score) และการกรองความเกี่ยวข้องภัยพิบัติ (Informativeness F1-Score) ข้าม 3 ยุคสมัยของการทำ Prompt Engineering:
1.  **V1 (Original Biased)**: Prompt ดั้งเดิมที่มีกฎเกณฑ์เชิงลบเชิงลบและเข้มงวดสูง (Strictness bias)
2.  **V2 (Optimized)**: ปลดล็อกข้อจำกัดทางสติปัญญาและลดเงื่อนไขที่ตึงตัวเกินไป 
3.  **V2-1 (V3 Decision Hierarchy)**: การใช้งานลำดับขั้นการตัดสินใจและการกำหนดกติกา Bias Rule เพื่อแก้ข้อจำกัดของ Sequential

## 2. บทวิเคราะห์เชิงประวัติศาสตร์และทางเทคนิค

### 1️⃣ วิกฤตการณ์เวอร์ชัน V1 (The V1 Bottleneck)
*   **ปัญหาหลัก**: ใน V1 เราพยายามบังคับทิศทางโมเดลด้วยคำว่า "ต้อง" หรือ "ห้าม" (เช่น Strictness Bias / Negative constraints) อย่างหนักหน่วง 
*   **ผลกระทบ**: ส่งผลเสียเชิงสถาปัตยกรรมอย่างรุนแรง โดยเฉพาะในกลุ่ม **Two-Layer Joint (Exp 2E)** ซึ่งโมเดลต้องวิเคราะห์ทั้งสองคำถามพร้อมกัน โทเค็นคำสั่งเชิงปฏิเสธสร้างภาระต่อความสนใจ (Attention Weights) ทำให้ Category F1 ร่วงลงไปอยู่ในระดับแย่มากคือ **`0.39 - 0.42`**

### 2️⃣ ก้าวกระโดดสู่เวอร์ชัน V2 (The V2 Optimization)
*   **การแก้ไข**: ถอนคำสั่งเชิงปฏิเสธที่กำกวมออกทั้งหมด เปลี่ยนเป็นการใช้นิยามแต่ละระดับอย่างเป็นมิตรต่อกระบวนการคิดของโมเดล
*   **ผลลัพธ์**: ประสิทธิภาพพุ่งขึ้นในทุกโมเดลและทุกโครงสร้าง โดยเฉพาะ **Two-Layer Joint (2E) ขยับขึ้นมาถึง +20% สำหรับ Typhoon** (ขึ้นไปแตะจุดสูงสุดเดิมคือ `0.6493`) และโมเดลอื่น ๆ ขยับพ้นเกณฑ์ `0.60` เป็นครั้งแรก

### 3️⃣ ความสมบูรณ์แบบทางโครงสร้างในเวอร์ชัน V2-1 (V3 Prompt Hierarchy)
*   **การแก้ไข**: 
    1.  นำหลักการ **Critical Decision Hierarchy** (ลำดับความสำคัญในการเลือกหมวดภัยพิบัติหากข้อมูลก้ำกึ่ง) มาเป็นแนวทางหลัก
    2.  แก้ปัญหาคอขวดสะสมความผิดพลาด (Error Propagation) ในโครงสร้างแบบ Sequential โดยใช้ **Aggressive Bias Rule** ใน Agent 1 ให้โมเดลปล่อยผ่านหากทวีตกล่าวถึงภัยพิบัติแม้จะเป็นการพยากรณ์อากาศหรือส่งกำลังใจ
*   **ผลลัพธ์**: 
    *   **ระบบ Sequential (Exp 3E) ประสบความสำเร็จอย่างยิ่งใหญ่**: Gemma-4 ขยับขึ้นจาก `0.5738` (V2) เป็น **`0.6430` (V2-1) (+6.92%)** และทำให้ระบบ Sequential คุ้มค่าน่าใช้งานในทันที
    *   **Gemma และ DeepSeek ในแบบ Joint (Exp 2E) พัฒนาขึ้นอีก +1% ถึง +2%** จากอานิสงส์ของขอบเขตข้อห้ามที่ชัดเจนขึ้น
    *   *ข้อควรระวัง:* สำหรับ Typhoon ความยาวของ Prompt V3 ส่งผลให้ประสิทธิภาพดร็อปลงเล็กน้อยในแบบ Flat และ Joint เนื่องจากขีดจำกัดความจำบริบทภาษาอังกฤษของโมเดลขนาดเล็ก
