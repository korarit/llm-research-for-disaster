## 2. การเปรียบเทียบผลลัพธ์โครงสร้าง Flat (1TH) vs. 2-Layer Joint (2TH)

ผลลัพธ์สรุปประสิทธิภาพความแม่นยำและการใช้โทเค็นเปรียบเทียบระหว่างสองสถาปัตยกรรม:

- **คะแนน F1-Score เฉลี่ยลดลงในบางโมเดล**:
  - **DeepSeek-v4-flash**: สูญเสียความแม่นยำในคลาสย่อยลงเล็กน้อย (Category F1 ลดลงจาก `0.7522` เหลือ `0.7283`)
  - **Gemma-4**: ได้ Category F1 ลดลงจากสูงสุดที่ `0.7557` เหลือ `0.7162`
  - *เหตุผล*: การบังคับให้โมเดลจำแนกและตอบกลับฟิลด์พร้อมกัน 2 เลเยอร์ สร้างภาระการประมวลผล (Cognitive Load) สูง ทำให้ความแม่นยำในการระบุหมวดหมู่ภัยพิบัติลดลง
- **Typhoon-v2.5 มีคะแนนเพิ่มสูงขึ้น (Improvement)**:
  - ค่า Category F1 ของ Typhoon เพิ่มขึ้นอย่างมีนัยสำคัญจาก `0.6870` (Flat) ไปเป็น **`0.7014`** (2-Layer Joint, T=0.1)
  - *เหตุผล*: โครงสร้างการแบ่งเลเยอร์ช่วยจำกัดขอบเขตคีย์เวิร์ดของภาษาไทยให้โมเดล Typhoon กรองความเกี่ยวข้องเบื้องต้นได้แคบลงก่อนนำไปคัดแยก ส่งผลเชิงบวกต่อโมเดลโมเลกุลเล็กของภาษาไทย
- **โทเค็นและเวลาประมวลผล (Overhead)**:
  - สถาปัตยกรรม Two-Layer ใช้ Token สูงขึ้นเฉลี่ย ~10-15% เนื่องจากต้องประมวลผลคำอธิบายฟิลด์ความเกี่ยวข้องใน System Instruction เพิ่มขึ้น ส่งผลให้ Latency ต่อโพสต์สูงขึ้นเฉลี่ย ~0.1 - 0.2 วินาทีในทุกโมเดล
