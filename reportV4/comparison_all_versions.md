# รายงานสรุปผลเปรียบเทียบทุกสถาปัตยกรรม (Flat vs. Two-Layer vs. 2-Agent) - ภาษาไทย

รายงานฉบับนี้สรุปเปรียบเทียบผลลัพธ์ประสิทธิภาพความแม่นยำ (F1-Score), ความเร็ว (Latency), และปริมาณการใช้โทเค็น (Token Usage) ระหว่าง 3 สถาปัตยกรรมในการคัดแยกข้อความภัยพิบัติภาษาไทย 1,000 แถว ได้แก่:
1. **Flat (Exp 1TH)**: จำแนกประเภทตรงขั้นตอนเดียว
2. **Two-Layer Joint (Exp 2TH)**: จำแนกพร้อมกันสองเลเยอร์ใน Function call เดียว
3. **2-Agent Sequential (Exp 3TH)**: จำแยกหน้าที่กรองความเกี่ยวข้องและแยกคลาสย่อยเป็น 2 เอเจนต์

## 3. บทวิเคราะห์และข้อสรุปทิศทางผลการทดลองในภาษาไทย

1. **สถาปัตยกรรม 2-Agent Sequential (Exp 3TH) ให้คะแนนสูงที่สุดสำหรับโมเดลภาษาไทย**:
   - การแยกบทบาทหน้าที่เป็นเอเจนต์ 2 ตัวที่ทำงานสอดรับกันอย่างเป็นอิสระ ช่วยจำกัดภาระทางความคิด (Cognitive Load) ของโมเดลได้ดีที่สุด ส่งผลเชิงบวกต่อโมเดลภาษาไทยอย่างชัดเจน โดยเฉพาะ **Typhoon-v2.5** ที่คะแนน F1 เพิ่มขึ้นถึง **+5.33%** ขึ้นมาอยู่ในระดับหัวแถวร่วมกับโมเดลใหญ่
2. **โมเดล Gemma-4 ตอบสนองดีที่สุดกับแบบ Flat (Exp 1TH)**:
   - สรุปได้ว่า Gemma-4 มีความสามารถระดับ Zero-shot ในภาษาไทยที่ดีเยี่ยมในขั้นตอนเดียว โดยสามารถจำแนกหมวดหมู่ได้อย่างเที่ยงตรงด้วยสถาปัตยกรรมแบบ Flat (สูงสุดถึง **`0.7557`**) แต่ได้ผลลัพธ์ลดลงเมื่อขยับเป็นสถาปัตยกรรมแบบอื่น
3. **การประเมินเพื่อการนำไปใช้งานจริง (Deployment Decision)**:
   - **ความแม่นยำสูงสุด**: แนะนำ **DeepSeek-v4-flash ร่วมกับสถาปัตยกรรม 2-Agent (Exp 3TH)** (F1 = `0.7583`)
   - **ความเร็วและประหยัดงบสูงสุด**: แนะนำ **Typhoon-v2.5 ร่วมกับสถาปัตยกรรม 2-Agent (Exp 3TH)** (F1 = `0.7403` แต่ตอบสนองเร็วที่ ~0.67s ต่อแถว)
