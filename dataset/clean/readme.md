# รายงานการวิเคราะห์ข้อมูลเบอร์โทรศัพท์และแหล่งข้อมูล (Phone Number & Source Analysis)

เบอร์โทรศัพท์สามารถใช้เป็นเครื่องระบุตัวตนบุคคล (Unique Identifier) ในการยื่นขอความช่วยเหลือจากอุทกภัยได้ ซึ่งช่วยให้เราสามารถวิเคราะห์พฤติกรรมการส่งคำขอซ้ำ (Duplicate Requests) ไปยังช่องทางต่าง ๆ ได้อย่างมีประสิทธิภาพ

---

## 1. ขนาดของข้อมูลในแต่ละแหล่ง (Data Source Sizes)

จากการประมวลผลไฟล์ `merged_clean.csv` (รวมทั้งหมด 1,861 คอมเมนต์) มีรายละเอียดของแต่ละแหล่งข้อมูลดังนี้:

| แหล่งข้อมูล (Source) | จำนวนคอมเมนต์ทั้งหมด | จำนวนคอมเมนต์ที่พบเบอร์โทร (มีสัดส่วน %) | จำนวนเบอร์โทรที่ตรวจพบทั้งหมด |
| :--- | :---: | :---: | :---: |
| **กองทัพบกทันกระแส** | 978 | 592 (60.53%) | 1,028 |
| **โลกใบใหญ่ของอองรี** | 516 | 269 (52.13%) | 456 |
| **พรรคประชาชน** | 367 | 185 (50.41%) | 339 |
| **รวมทั้งหมด (Merged)** | **1,861** | **1,046 (56.21%)** | **1,823** |

---

## 2. การวิเคราะห์การซ้ำกันของข้อมูล (Duplication Analysis)

จากการตรวจจับเบอร์โทรศัพท์ 10 หลัก (รวมถึงการแปลงรูปแบบรหัสประเทศ `+66` / `66` ให้เป็นเลข 10 หลักขึ้นต้นด้วย `0`) พบสถิติดังนี้:

- **จำนวนเบอร์โทรทั้งหมดที่ตรวจเจอ (Total Occurrences):** 1,823 ครั้ง
- **จำนวนเบอร์โทรที่ไม่ซ้ำกันเลย (Unique Phone Numbers):** 1,700 เบอร์
- **จำนวนครั้งที่ซ้ำกัน (Total Duplicates):** **123 ครั้ง**
  *(คำนวณจาก: 1,823 ครั้งที่ตรวจพบ - 1,700 เบอร์ที่ไม่ซ้ำกัน)*
- มีเบอร์โทรศัพท์จำนวน **101 เบอร์** ที่พบการปรากฏซ้ำมากกว่า 1 ครั้งขึ้นไป

---

## 3. การส่งคำขอซ้ำข้ามแหล่งข้อมูล (Cross-Source Duplicates)

ในจำนวนเบอร์โทรศัพท์ที่ไม่ซ้ำกันทั้งหมด 1,700 เบอร์ มีเบอร์โทรศัพท์จำนวน **49 เบอร์** ที่มีการส่งคำขอความช่วยเหลือซ้ำข้ามช่องทาง (ข้ามแหล่งข้อมูล) แสดงให้เห็นว่าผู้ประสบภัยคนเดียวกันทำการส่งข้อมูลไปยังเพจ/ช่องทางต่าง ๆ มากกว่าหนึ่งแห่งเพื่อเพิ่มโอกาสในการเข้าช่วยเหลือ โดยแบ่งออกได้เป็น:

- **ซ้ำระหว่าง กองทัพบกทันกระแส และ โลกใบใหญ่ของอองรี:** 26 เบอร์
- **ซ้ำระหว่าง กองทัพบกทันกระแส และ พรรคประชาชน:** 19 เบอร์
- **ซ้ำระหว่าง พรรคประชาชน และ โลกใบใหญ่ของอองรี:** 4 เบอร์

*(หมายเหตุ: ไม่มีเบอร์โทรศัพท์ใดที่ถูกตรวจพบพร้อมกันครบทั้ง 3 ช่องทาง)*

## 4. สคริปต์ที่ใช้ประมวลผล (Processing Scripts)

เพื่อความโปร่งใส สคริปต์สำหรับการทำความสะอาด คัดแยก และตรวจจับเบอร์โทรศัพท์ถูกรวบรวมและย้ายมาไว้ที่โฟลเดอร์นี้ทั้งหมด:

1. [clean_dataset.py](file:///e:/nlp-for-disaster/dataset/clean/clean_dataset.py) - ทำความสะอาดไฟล์ข้อมูลดิบ (ลบแถวที่ไม่มีข้อความออก และใส่ ID เริ่มต้น)
2. [merge_datasets.py](file:///e:/nlp-for-disaster/dataset/clean/merge_datasets.py) - รวมไฟล์ทั้งหมดเข้าด้วยกัน ลบแถวที่เนื้อหาข้อความซ้ำกัน และจัดระเบียบ ID ใหม่
3. [detect_phone_numbers.py](file:///e:/nlp-for-disaster/dataset/clean/detect_phone_numbers.py) - ค้นหาเบอร์โทรศัพท์ด้วย regex, ทำการ normalize เป็นตัวเลข 10 หลัก และสร้างรายงานความซ้ำของเบอร์โทรศัพท์

*สคริปต์ทั้งหมดถูกออกแบบให้สามารถรันจากไดเรกทอรีใดก็ได้ (Path-Independent) ตัวอย่างเช่น:*
```bash
python dataset/clean/clean_dataset.py
python dataset/clean/merge_datasets.py
python dataset/clean/detect_phone_numbers.py
```
