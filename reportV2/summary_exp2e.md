# รายงานวิเคราะห์เปรียบเทียบผลการทดลอง: Experiment 2E (Two-Layer Joint)

รายงานฉบับนี้ทำการวิเคราะห์เปรียบเทียบผลลัพธ์ของโมเดล `deepseek-v4-flash`, `typhoon-v2.5` และ `gemma-4` ในการทดลอง **Exp 2E (Biased Two-Layer Joint)** ในการรันใหม่เทียบกับ Baseline ใน **reportV1** ครอบคลุมอุณหภูมิ 0.0 - 0.3

การทดลองแบบ Two-Layer Joint เป็นการแบ่งการทำนายออกเป็น 2 ขั้นตอน (ยิง API 2 รอบ):
1. ขั้นกรองความเกี่ยวข้อง (Informativeness Filter)
2. ขั้นระบุหมวดหมู่ภัยพิบัติเฉพาะ (Category Classification - ยิงเฉพาะรายการที่สอบผ่านขั้นแรก)

## 2. การวิเคราะห์ความแตกต่างและข้อค้นพบสำคัญ

### 1. 🚀 ประสิทธิภาพดีขึ้นอย่างมหาศาล (จูนปลดล็อกระบบกรอง)
* ในรอบ Baseline เดิม ทุกโมเดลมีผลลัพธ์แย่มากจากการโดน "Strictness Bias" บีบให้ปัดทอนทิ้งเป็นขยะตั้งแต่ขั้นแรก
* ในรอบรันใหม่ F1-score ด้านการกรองความเกี่ยวข้องทะยานขึ้นไปแตะระดับ **`~0.88 - 0.90`** ในทุกโมเดล (เช่น **Gemma-4** ได้รับคะแนน Info F1 เพิ่มขึ้นถึง **+34.7%** absolute!)
* คะแนนจำแนกหมวดหมู่ (Category F1) ขยับสูงขึ้นถ้วนหน้าเฉลี่ย **+11% ถึง +24%** ขยับขึ้นมาแตะระดับ `~0.60 - 0.64` ในทุกตัวแปร

### 2. 🏆 จุดเด่นสำคัญ: ทุบสถิติ F1-Score สูงสุดของโครงการ
* **`typhoon-v2.5` ที่ Temp 0.2** ทำคะแนน Category F1 ได้สูงสุดถึง **`0.6493`** ซึ่งเป็น**คะแนนการจำแนกหมวดหมู่ที่สูงที่สุดนับตั้งแต่เริ่มโปรเจกต์นี้มา**
* แสดงให้เห็นว่าการแบ่งภาระงานแยกออกเป็น 2 ชั้นอย่างอิสระ (Two-Layer Joint) ช่วยให้โมเดลประมวลผลหมวดหมู่ย่อยได้ละเอียดและแม่นยำขึ้นเมื่อตัดระบบคำสั่งที่ตึงเกินไปออกแล้ว

### 3. 📉 ประหยัด Token ไปได้เฉลี่ย 16% - 18%
